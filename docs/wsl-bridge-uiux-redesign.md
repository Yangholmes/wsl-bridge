# WSL Bridge UI/UX 重新设计方案

## 1. 设计理念

### 1.1 核心原则
- **现代控制台美学**：保持工程控制台的专业性，通过现代化设计语言提升视觉吸引力
- **信息层次清晰**：通过视觉层次引导用户注意力，关键信息一目了然
- **流畅交互体验**：减少认知负担，提高操作效率
- **视觉一致性**：统一的设计语言贯穿整个应用

### 1.2 设计风格
- **风格定位**：现代科技控制台
- **视觉特征**：
  - 卡片式布局增加层次感
  - 精致的阴影和圆角
  - 清晰的状态指示
  - 微妙的动画过渡
  - 高对比度的信息展示

## 2. 布局重构

### 2.1 新布局结构

```
┌────────────────────────────────────────────────────────────────┐
│  Logo  |  应用状态指示器  |  规则引擎状态  |  快捷操作按钮组  |  设置 │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │  导航标签栏  │  │              │  │              │       │
│  │  (顶部导航)  │  │   主内容区   │  │   快捷面板   │       │
│  └──────────────┘  │              │  │  (可选显示) │       │
│                    │              │  │              │       │
│  ┌──────────────┐  │              │  └──────────────┘       │
│  │  状态卡片    │  └──────────────┘                          │
│  └──────────────┘                                             │
│                                                                │
├────────────────────────────────────────────────────────────────┤
│  应用版本  |  内核版本  |  最后应用时间  |  网络状态  |  用户信息 │
└────────────────────────────────────────────────────────────────┘
```

### 2.2 布局特点

1. **顶部导航栏**（高度 64px）
   - Logo 和应用名称
   - 应用状态指示器（大型、醒目）
   - 规则引擎状态徽章
   - 快捷操作按钮组（应用规则、停止规则、扫描拓扑）
   - 设置入口

2. **导航标签栏**（高度 56px）
   - 采用顶部标签导航，替代左侧导航
   - 标签：概览、规则、拓扑、运行状态、日志
   - 当前标签高亮显示
   - 支持徽章显示（如错误数量）

3. **主内容区**
   - 根据不同页面自适应布局
   - 支持分栏布局（主内容 + 快捷面板）
   - 响应式设计

4. **底部状态栏**（高度 40px）
   - 应用版本信息
   - 网络状态指示
   - 最后操作时间
   - 用户信息

│  │  (大型)     │  │  (大型)     │  │  (大型)     │        │
│  └─────────────┘  └─────────────┘  └─────────────┘        │
│                                                            │
│  ┌──────────────────────────────────────────────────────┐ │
│  │  快速操作区                                          │ │
│  │  [新建规则]  [扫描拓扑]  [查看错误日志]  [导出配置] │ │

│  ┌─────────────────────────┐  ┌────────────────────────┐ │
│  │  最近活动               │  │  运行中规则            │ │

#### 设计要点
1. **状态卡片**
   - 大型卡片，高度 160px
   - 醒目的状态指示器（图标 + 文字）
   - 关键数据大号显示
   - 微妙的渐变背景或阴影
   - 悬停时有轻微的上浮动画

2. **快速操作区**
   - 一行排列的操作按钮
   - 图标 + 文字组合
   - 主要操作使用品牌色
   - 次要操作使用次要色

3. **最近活动**
   - 时间线设计
   - 显示最近的操作记录
   - 不同操作类型用不同颜色标识

4. **运行中规则**
   - 紧凑的列表视图
   - 显示规则名称、状态、类型
   - 快速启停按钮

## 5. 组件设计规范

### 5.1 状态徽章（Status Badge）

#### 设计规范
- **尺寸**：高度 24px，内边距 8px 水平
- **圆角**：radius-full (999px)
- **字体**：text-sm, font-medium
- **状态颜色**：
  - 运行中（Running）：ok-600 背景 + 白色文字
  - 已停止（Stopped）：gray 背景
  - 错误（Error）：danger-600 背景 + 白色文字
  - 未知（Unknown）：gray 背景

### 5.2 按钮（Button）

#### 类型
1. **主要按钮（Primary）**
   - 背景：brand-600
   - 文字：白色
   - 悬停：brand-700
   - 高度：40px

2. **次要按钮（Secondary）**
   - 背景：transparent
   - 边框：border-default
   - 文字：text-primary
   - 悬停：bg-canvas

3. **危险按钮（Danger）**
   - 背景：danger-600
   - 文字：白色
   - 悬停：darker shade

4. **图标按钮（Icon）**
   - 尺寸：32px x 32px
   - 圆角：radius-md

### 5.3 卡片（Card）

#### 设计规范
- **背景**：bg-card
- **圆角**：radius-lg (12px)
- **阴影**：shadow-md
- **内边距**：space-24 (24px)
- **悬停效果**：shadow-lg + transform: translateY(-2px)

### 5.4 输入框（Input）

#### 设计规范
- **高度**：40px
- **圆角**：radius-md (8px)
- **边框**：border-default
- **聚焦**：border-brand + shadow-sm
- **错误**：border-danger

### 5.5 表格（Table）

#### 设计规范
- **表头**：背景 bg-panel, 字体 font-semibold, 颜色 text-secondary
- **行高**：48px
- **悬停**：bg-canvas
- **选中**：brand-50 背景
- **边框**：border-default
